# Changelog

## 0.2.0

- 在线 Demo 优先把每日真实仓库快照镜像到可写运行库，避免云端残留的旧模拟库遮蔽新快照；快照异常时才显式回退模拟数据。
- 修正 SQLGlot 仓库路径，并在单个 PR Review 资源暂时消失时继续刷新其余真实数据。
- 记录每类 GitHub 数据的分页、时间范围、刷新时间和历史完整性。
- 增加数据新鲜度、最近采集失败和 PR Review 窗口覆盖提示。
- 增加维护者待办，定位陈旧、无人响应或等待 Review 的具体 Issue/PR。
- 使用编号 `schema_migrations` 事务化升级旧 DuckDB 快照。
- GitHub 5xx 与临时网络错误采用有界指数退避重试；限流仍立即返回明确错误。
- CLI 摘要与 HTML 报告同步输出数据可信度和维护待办结论。
