# 指标字典

所有时间在数据库中统一存储并按 UTC 计算。页面展示的百分比保留一位小数。

除特别说明外，每个指标都支持时间范围筛选（30/90/180 天及自定义日期），
窗口作用于实体的 `created_at`（Issue/PR）或 `committed_at`（Commit）。

| 指标 | 定义 | 注意事项 |
|---|---|---|
| Issue 关闭率 | 已关闭 Issue 数 ÷ 窗口内已采集 Issue 总数 | 较新的 Issue 会降低该值 |
| Issue 中位关闭时间 | `median(closed_at - created_at)` | 仅统计已关闭 Issue |
| Issue P90 关闭时间 | 90% 已关闭 Issue 的关闭时长不超过该值 | 对长尾积压比平均数更敏感 |
| Issue 首次响应时间 | Issue 创建到第一条非作者、非 Bot 评论的时长 | Bot（dependabot / `[bot]` / codecov 等）和作者自评被排除 |
| 无响应 Issue | 窗口内没有任何合格回复的 Issue 数 | 单独计数，不参与中位数计算，避免用 0 拉低分布 |
| 陈旧开放 Issue | 状态为 open 且创建超过 90 天 | 以窗口结束日为基准；阈值可在后续版本配置化 |
| 30/90 天积压比例 | 开放 Issue/PR 中创建超过 30/90 天的占比 | 以窗口结束日为基准 |
| PR 合并率 | 已合并 PR ÷ 已结束的非草稿 PR | 排除开放 PR 和草稿 PR |
| PR 中位合并时间 | `median(merged_at - created_at)` | 仅统计已合并 PR |
| PR 首次 Review 时间 | PR 创建到第一条非作者、非 Bot Review 的时长 | 只覆盖最近 180 天内创建的 PR（采集成本控制） |
| 无 Review PR | 窗口内没有任何合格 Review 的 PR 数 | 同样单独计数，不参与中位数 |
| 90 天活跃贡献者 | 最近 90 天内提交 Commit 或创建 PR 的去重用户 | Bot 暂未自动剔除；该指标始终以“当前”为基准，不受窗口 start 影响 |
| 头部贡献者活动占比 | 活动最多用户的 Commit 与 PR 数 ÷ 全部用户活动数 | 是集中度代理指标，不等同于严格 Bus Factor |
| 月度贡献者留存 | 某 Cohort 在第 N 月仍有 Commit 或 PR 的用户数 ÷ Cohort 人数 | 受采集窗口左截断影响 |

## 数据可信度口径

每次采集都会为 Issue、Pull Request、Commit、Release、Issue 评论和 PR Review
记录页数、页数上限、记录时间范围和刷新时间。

- 若某次采集的最后一页仍有 100 条记录并达到 `max_pages`，该实体标记为“本次截断”。
- 首次或增量采集一旦发生截断，`history_complete` 会保守地保持为 false；后续较小的增量更新不能证明此前缺口已经补齐。
- Release 每次全量刷新，因此一次未截断的全量刷新可以恢复其完整状态。
- PR Review 的覆盖范围固定为最近 180 天内创建的 PR；页面会将这一成本边界单独提示，不把它伪装成完整历史。
- 仓库元数据超过 48 小时未刷新时触发新鲜度提示；最近一次采集失败时触发高优先级质量告警。

## 维护者待办规则

待办列表只包含仍然开放、且已经达到处理阈值的对象：

- 开放超过 90 天的 Issue：高优先级。
- 超过 30 天且没有合格响应的 Issue：高优先级。
- 超过 7 天仍无响应的 Issue：中优先级。
- 开放超过 90 天的非草稿 PR：高优先级。
- 超过 14 天仍无合格 Review 的 PR：高优先级。
- 超过 7 天仍无 Review，或开放超过 30 天的 PR：中优先级。

响应与 Review 继续排除作者自评和常见 Bot。列表按优先级和等待时间排序，最多展示 20 项，
目的是给维护者提供可直接打开的工作清单，而不是替代仓库自己的优先级判断。

## 多仓库对比口径

对比视图取每个仓库在相同时间窗口下的关闭率、合并率、各自中位耗时、活跃贡献者数和集中度。
不同仓库的活动量基数不同，因此对比以比率和中位数为主，避免直接比绝对数量。

## 风险规则

- 开放 Issue 中超过 90 天的占比达到 40%：高风险积压提示。
- 单一贡献者活动占比达到 50%：高集中度提示。
- PR 合并率低于 40%：中风险转化提示。

规则用于快速发现值得调查的方向，不构成对项目质量的最终评分。

## 报告生成规则

分析报告基于上述指标用阈值规则归纳“主要发现、风险、建议”，例如：
PR 中位合并 > 3 天 → 建议优化 Review 流程；无响应比例 ≥ 20% → 建议定期巡检；
积压占比 ≥ 40% → 建议做积压清理；集中度 ≥ 50% → 建议 Bus Factor 风险。
报告本身不引入模型预测，所有结论可追溯到具体指标值。
